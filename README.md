# 뉴스 및 소셜 데이터를 활용한 AI 입법 수요 분석 서비스 모델 개발

## 1. 프로젝트 개요 
본 프로젝트는 방대한 양의 뉴스 및 소셜 데이터를 분석하여 **잠재적인 입법 수요를 탐지**하고, 각 수요의 사회적 중요도와 사회적 파급력 및 시급성을 평가하는 AI 서비스 모델을 개발하는 것을 목표로 한다.

## 2. 주요 기능
### news_model
- 데이터 분석 및 전처리 
    - 계층적 데이터 분할: 데이터셋을 학습용과 평가용으로 분리할 때, 중분류(MAIN_NAME)의 분포를 동일하게 유지하여 모델이 편향되지 않게 학습하도록 한다.
    - Sliding Window Tokenization : BERT 모델의 최대 입력 길이(512 토큰)를 초과하는 장문의 뉴스 기사도 정보 손실 없이 처리하기 위해, 텍스트를 겹치는 구간을 두고 여러 조각으로 나누어 분석한다.
    - 클래스 불균형 해소(WeightedRandomSampler): 학습 데이터에서 적게 등장하는 소수 클래스도 모델이 충분히 학습할 수 있도록, 각 클래스의 중요도에 따라 가중치를 부여하여 샘플링한다.

- 계층적 분류 모델 
    - 고성능 한국어 분류 모델 사용 : 한국어 문맥 이해 능력이 뛰어난 klue/roberta-base 모델을 기반으로 뉴스 기사를 효과적으로 분류한다.
    - 계층적 분류 구조 : 대분류(category)를 먼저 예측하고, 그 예측 결과를 바탕으로 연관된 중분류(MAIN_NAME)을 예측하는 2단계 분류구조를 가진다.
    - 논리적 오류 방지 마스크 : 예측된 대분류와 상관없는 중분류가 선택되는 오류를 방지하는 허용 마스크를 적용해 예측의 신뢰성과 정확성을 높인다.
    - 문서 단위 최종 판정 : 슬라이딩 윈도으로 인해 여러 청크로 나뉜 텍스트의 예측 확률들을 다시 하나의 문서 단위로 통합하여, 가장 연관성이 높은 최종 대분류와 중분류를 결정한다. 

### social_model
- 데이터 분석 및 전처리 
    - 템블릿 블랙리스트 생성 : 소셜데이터 특성상 광고,특정 도메인 양식과 같은 글의 문맥 및 찬/반을 분류할 때 불필요한 텍스트 전처리 수행한다.
    - 광고 유형의 글 제거(정규식) : 광고 유형의 글들은 데이터에서 제거한다.

- 정책/법안 입장 분류
    - 최종 목표: 텍스트에 담긴 입장을 '찬성_개정/강화', '찬성_폐지/완화', '반대_현상유지', '중립'의 4가지 클래스로 분류한다.
    - 계층적 분류 모델 
        - 1단계 (논조 분류): '찬성', '부정', '중립'의 큰 틀로 먼저 분류한다.
        - 2단계 (부정 입장 세분화): 1단계에서 '부정'으로 판단된 텍스트에 한해, '찬성_폐지/완화'와 '반대_현상유지'를 구분하는 모델을 적용하여 분류한다.

### score_model 
- 탐지된 입법 수요에 대해 다각적인 평가지표(확산성, 언급량, 찬반여부)를 종합하여 중요도 점수를 산출한다.

## 3. 폴더 구조 
```
AIDA/
├── README.md               # 프로젝트 설명 및 사용 가이드
├── data/
│   ├── raw/                # 원본 데이터
│   ├── processed/          # 전처리 완료된 데이터
│   └── split/              # 모델 학습/검증/테스트용으로 분할된 데이터
├── models/                 # 학습된 모델 파일 저장 폴더
├── notebooks/              # 데이터 분석 및 모델 실험용 Jupyter Notebook 파일
├── scripts/                # 실행 자동화를 위한 쉘 스크립트
├── src/                    # 핵심 소스 코드 (.py)
├── demo/                   # 모델 시연 영상
└── figures/                # README 및 보고서에 사용된 이미지
```
